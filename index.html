01 — Introduction

What is Vortex

Vortex is a desktop threat analysis tool designed for the Discord community. It helps users identify malicious executables — specifically files that steal Discord tokens, session data, and credentials — before those files are opened or shared.

Multi-Layer Analysis

Five independent analysis methods run simultaneously — static inspection, behavioral monitoring, cloud threat intelligence, sandbox data, and AI verdict synthesis.

Discord-Focused Detection

Purpose-built to catch Discord token stealers, webhook exfiltrators, and credential grabbers — threats that generic scanners frequently miss or misclassify.

Global Threat Database

Every completed scan contributes to a shared database. When any user scans a previously analyzed file, cached results return instantly — protecting the whole community.

Core Purpose

Built for community-level defense

Discord's ecosystem is a frequent target for social-engineering attacks where malicious files disguised as game cheats, Nitro generators, or free software are shared in servers. Vortex exists to give users a fast, reliable way to verify a file's intent before it can do harm.

  • Detects token stealers that bypass traditional antivirus
  • Produces human-readable reports with evidence reasoning
  • Community leaderboard tracks top threat hunters

02 — Getting Started

Create Your
Account

Account creation happens inside the Vortex application. You need one account to use the tool. Accounts are free, each person is allowed exactly one, and registration must be done on your real home or mobile network.

1

Open Vortex and click Register

Launch Vortex for the first time. On the login screen, click the Register tab. Do not use a VPN, proxy, or any non-residential IP during this step — registration from datacenter IPs is blocked.

2

Fill in your account details

Enter your chosen username, a valid email address, and a password. Your username is permanent — it cannot be changed after registration, so choose carefully. The email is used for login only.

Usernames cannot be changed once set. Think carefully before registering.
3

Account is bound to your hardware

On first login, your account is permanently linked to your device's hardware ID. This prevents account sharing. If you need to transfer your account to a new device later, contact the Vortex team directly. Attempting to bypass this creates a new bannable account.

4

You're in — configure your API keys next

After successful registration and login, your account is live. Before scanning, you'll need to add your API keys to config.json. Head to the API Keys section of this guide for the exact steps.

One person, one account. Creating multiple accounts for the same person or device is a bannable offense with no appeal pathway. The system monitors for duplicate hardware IDs automatically.

API Keys

Vortex requires three external API keys to unlock its full analysis capability. Keys are stored locally in config.json — they never leave your machine except to query their respective services during a scan.

Free Key

4. Paste it into config.json as vtApiKey

05 — Scanning

How to Scan a File

Three methods to load a file into Vortex. All three lead to the identical analysis pipeline.

File Browser

Click the upload zone in the Scanner tab. A standard file picker opens — navigate to the target file and click Open to begin analysis.

  • Click the upload zone on the Scanner tab
  • Navigate to the file in the dialog
  • Click Open — scan begins automatically

Drag & Drop

Drag any file from Windows File Explorer and drop it directly onto the Vortex window. The fastest method — zero navigation required.

  • Open File Explorer alongside Vortex
  • Drag the target file
  • Drop onto the Vortex window — instant load

Folder or Archive

Select an entire folder or a RAR archive. Vortex enumerates every file inside and scans each individually through the full pipeline.

  • Browse to the folder or .rar file
  • Each contained file is scanned one by one
  • Each file costs a separate scan credit
What Happens During a Scan
1

File is hashed — database checked

Vortex computes a cryptographic hash of the file. If this exact file has been scanned before by any user in the global database, the cached result is returned immediately. Your scan credit is still used — but the result is instant.

2

Static analysis begins immediately

The file is inspected without being executed. Internal structure, byte patterns, embedded strings, import tables, entropy levels, compression signatures, and metadata are all analyzed. This runs on every file, every time.

3

Cloud intelligence queries run in parallel

Your VirusTotal and Hybrid Analysis keys are used to query existing reports. These run simultaneously alongside local analysis to minimize total scan time. For exe/dll/sys files, both services must return results for the scan to be marked complete and saved.

4

Behavioral analysis observes runtime behavior

For supported executable formats, the file is monitored in a controlled, instrumented environment. This layer observes what the file actually does when it runs — file system access, network connections, memory operations, code injection, and attempts to access Discord-related data.

5

AI engine produces the final verdict

All gathered data is fed to multiple AI models. Each independently analyzes the full evidence set and produces a verdict with a confidence score. A final synthesis pass combines all judgments into the definitive result — with reasoning you can read in the report.

New exe/dll: 60–180 seconds · Duplicate files: instant · Other formats: faster

  • Stack on top of your daily free allocation
  • Purchased separately — contact the team for pricing

08 — Scanning

Analysis
Methods

Vortex runs five independent analysis layers and combines their outputs into a weighted composite risk score. Each layer targets a different class of evidence.

1
Static Analysis Always runs · No API key needed

The file is inspected without execution. Internal structure, byte patterns, embedded strings, import tables, entropy levels, compression signatures, and PE metadata are all analyzed. Runs on every file, every time. This layer runs first and produces the foundational evidence set for all subsequent layers.

2
Behavioral Analysis Always runs on supported formats

For supported executable formats, the file is monitored in a controlled, instrumented environment. This layer observes what the file actually does when it runs — file system access, network connections, memory manipulation, code injection attempts, and any access to Discord-related data or processes. This is the most reliable method for detecting evasive threats that appear harmless in static analysis.

3
VirusTotal Cross-Reference Requires your VT API key

Queries VirusTotal's database using the file's SHA-256 hash to retrieve results from 70+ antivirus engines. Vortex collects how many engines flagged the file as malicious, what labels they assigned, and specific detection names. This provides immediate broad-coverage verdict data that supplements local analysis with community-wide knowledge.

4
Sandbox Intelligence — Hybrid Analysis Requires your HA API key

Hybrid Analysis maintains a large database of files that have been previously sandboxed. Vortex queries this database by file hash to retrieve existing behavioral analysis reports — including threat tags, network behavior summaries, verdicts, and classification labels. This provides behavioral context for files that may not yet trigger static signatures, without re-running the sandbox. Required for exe/dll scan completeness.

5
AI Verdict Engine Requires your OpenRouter key

Multiple AI models independently receive all gathered evidence — static findings, behavioral observations, cloud verdicts, and more — and each produces a verdict with a confidence score and written reasoning. A final AI synthesis pass then reviews all prior model judgments and produces the definitive classification. The AI pipeline is specifically tuned to differentiate Discord token stealers from software categories that exhibit similar low-level behaviors, like game cheats.

Every complete scan is stored in the global database and permanently linked to your account. View your full history from the History tab inside Vortex.

The History tab shows every scan you've submitted — file name, hash, risk score, detection label, timestamp, and completeness status. Click any entry to open the full detailed report with evidence from all analysis layers.

Global Database

Once a file is scanned and marked complete, it enters the shared global database. Any user scanning that same file gets the cached result instantly. The user who first submitted the file is permanently credited as the original discoverer.

Only complete scans are permanently stored. Incomplete scans — where cloud services timed out — are not saved and will not appear in your history. Re-scan the file to get a complete, permanent result.

14 — Reference

Rules &
Conduct

Vortex enforces strict policies to maintain platform integrity and prevent abuse. Violations are detected automatically and result in permanent bans with no appeal pathway.

Prohibited

  • Creating more than one account per person or device
  • Using VPNs or proxies during account registration
  • Sharing your account, session, or credentials with others
  • Attaching debugging tools to the Vortex process
  • Intercepting, tampering with, or modifying Vortex network traffic
  • Attempting to reverse-engineer or circumvent security measures
  • Submitting false, fabricated, or manipulated scan data

Permitted

  • Scanning any file you have a legitimate reason to analyze
  • Using a VPN after registration — login and scanning are fine
  • Sharing scan result summaries and reports in communities
  • Using all 5 of your free daily scans every day
  • Discussing Vortex findings and results publicly
  • Reporting bugs or suspected false positives to the team
Vortex actively monitors for policy violations during all sessions. Violations result in the immediate and permanent ban of the account, device, and registration IP. No manual review, no appeal, no exceptions.
Best Practices

Configure all three API keys before your first scan

Without VirusTotal and Hybrid Analysis keys, executable scans never reach completion status and results aren't saved. Without your OpenRouter key, the AI verdict layer is disabled. Set up all three in config.json before scanning.

Never close Vortex or disconnect mid-scan

Closing the app or losing internet connectivity during a scan wastes your credit without producing a saved result. Let scans run to completion — especially the cloud query phase, which can take 30–90 seconds depending on server response times.

Protect your config.json file

Your API keys live in config.json. Never share this file, include it in screenshots, or upload it anywhere. Anyone with your keys can consume your API quota and run scans under your identity.

Prioritize uncertain files with your free scans

With 5 free scans daily, focus on files you genuinely need to analyze. Don't waste free scans on files you already know are clean. Save premium credits for batch scanning sessions where you're processing many new files.
